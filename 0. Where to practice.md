# Where to practice Docker

You have several good options.

### Option 1 (Recommended): Install Docker Engine on Linux in a Virtual Machine

If your PC can run a lightweight Linux VM, this is the closest experience to a real production server.

- Install a virtualization software like VirtualBox (if your PC supports it).
- Install Ubuntu Server (minimal installation).
- Install Docker Engine (not Docker Desktop).
- Practice all Docker commands inside the VM.

**Advantages:**

- Exactly what you'll use on most VPS servers.
- No Docker Desktop required.
- Great for deployment learning.

**Disadvantage:**

- Needs enough RAM (at least 4 GB total system RAM is preferable).

Alternatively, you can install Docker Engine Directly on Linux (Dual Boot). This gives the best performance.

### Option 2 (VPS is for Production): Use a Cloud Linux Server

You can rent a cheap VPS for around $4–6/month (or use a free trial if available).

**Examples:**

- DigitalOcean
- Hetzner
- Vultr
- Oracle Cloud Free Tier (if available in your region)

*SSH into the server and install Docker:*

```bash
sudo apt update
sudo apt install docker.io
```

### Option 3 (Great for learning): Use GitHub Codespaces

GitHub Codespaces provides an online VS Code environment where Docker is available.

**Pros:**

- Nothing to install.
- Runs in your browser.
- Great for learning.

**Cons:**

Free usage is limited.

### Option 4: Use an Online Linux Terminal

Websites like:

- Killercoda
- Katacoda-style tutorials (where available)

provide interactive Docker labs directly in the browser.

# Using GitHub Codespaces

This is the fastest way to start.

1. Create a GitHub repository.
2. Upload your Django project.
3. Click **Code → Codespaces → Create codespace on main**.
4. Wait for the environment to start.
5. Open the terminal.

Check if Docker is available:

```bash
docker --version
docker compose version
```

Create a Dockerfile (starting point for docker) and open it:

```bash
touch Dockerfile
code Dockerfile
```

Paste something like this for a Django project in the Dockerfile:

```dockerfile
FROM python:3.12-slim

WORKDIR /app

COPY requirements.txt .

RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 8000

CMD ["python", "manage.py", "runserver", "0.0.0.0:8000"]
```

Then you can build and run docker container:

```bash
docker build -t mysite .
docker run -p 8000:8000 mysite
```

No software needs to be installed on your PC.

---

If your Dockerfile is in another folder, for example:

```
Docker-Pratice/
    backend/
        Dockerfile
```

Then run:

```bash
docker build -t mysite backend
```

or

```bash
cd backend
docker build -t mysite .
```

# Using Linux VPS (Recommended)

If you already know basic Linux commands, this is the best learning environment.

### Step 1: Rent a VPS

A typical starter VPS:

* 1 vCPU
* 1 GB RAM
* Ubuntu 24.04 LTS

### Step 2: Connect

From Windows PowerShell:

```bash
ssh username@your_server_ip
```

### Step 3: Install Docker

```bash
sudo apt update
sudo apt install docker.io docker-compose-v2 -y

sudo systemctl enable docker
sudo systemctl start docker
```

Verify:

```bash
docker --version
docker compose version
```

### Step 4: Clone your Django project

```bash
git clone https://github.com/yourname/project.git

cd project
```

### Step 5: Create a Dockerfile

Example:

```dockerfile
FROM python:3.12-slim

WORKDIR /app

COPY requirements.txt .

RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 8000

CMD ["python", "manage.py", "runserver", "0.0.0.0:8000"]
```

### Step 6: Create a `docker-compose.yml`

```yaml
services:
  web:
    build: .
    ports:
      - "8000:8000"
```

### Step 7: Run

```bash
docker compose up --build
```

Visit:

```
http://your_server_ip:8000
```
