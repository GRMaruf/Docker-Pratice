# Step 10: Deploying to a Linux VPS

We'll deploy this architecture:

```text
                Internet
                     │
                     ▼
              Nginx (Container)
                     │
                     ▼
            Gunicorn (Container)
                     │
                     ▼
          Django Application
                     │
                     ▼
      PostgreSQL (Container)
                     │
                     ▼
          Docker Volume (Persistent Data)
```

---

# Step 1: Get a Linux Server

Typical choices include:

* Ubuntu Server 24.04 LTS
* Debian 12

A small VPS with:

* 1 vCPU
* 1–2 GB RAM
* 20–25 GB SSD

is enough for learning and small projects.

---

# Step 2: Connect

From Windows PowerShell:

```bash
ssh username@your_server_ip
```

---

# Step 3: Update the Server

```bash
sudo apt update
sudo apt upgrade -y
```

---

# Step 4: Install Docker

```bash
sudo apt install docker.io docker-compose-v2 git -y
```

Enable Docker:

```bash
sudo systemctl enable docker
sudo systemctl start docker
```

Verify:

```bash
docker --version
docker compose version
```

---

# Step 5: Clone Your Project

```bash
git clone https://github.com/your-username/your-project.git
```

```bash
cd your-project
```

---

# Step 6: Upload the `.env`

Never store production secrets in Git.

Create the file on the server:

```bash
nano .env
```

Example:

```text
DEBUG=False

SECRET_KEY=<your-long-random-secret>

POSTGRES_DB=production_db
POSTGRES_USER=production_user
POSTGRES_PASSWORD=<strong-password>

DB_HOST=db
DB_PORT=5432

ALLOWED_HOSTS=your-domain.com,www.your-domain.com
```

---

# Step 7: Build Everything

```bash
docker compose up --build -d
```

Docker will:

* Build Django
* Pull PostgreSQL
* Pull Nginx
* Create the network
* Create the volume
* Start the containers

---

# Step 8: Run Migrations

```bash
docker compose exec web python manage.py migrate
```

Create an admin:

```bash
docker compose exec web python manage.py createsuperuser
```

---

# Step 9: Collect Static Files

```bash
docker compose exec web python manage.py collectstatic --noinput
```

Nginx can now serve the collected files directly.

---

# Step 10: Check Everything

View running containers:

```bash
docker compose ps
```

View logs:

```bash
docker compose logs -f
```

---

# Step 11: Configure a Domain

Suppose you own:

```text
example.com
```

In your DNS provider, add an **A record**:

```text
example.com → Your VPS IP
```

After DNS propagates, your domain points to your server.

---

# Step 12: Enable HTTPS

The common production setup is:

```text
Internet
     │
 HTTPS (443)
     │
     ▼
Nginx
     │
 HTTP (internal)
     │
     ▼
Gunicorn
```

A popular way to obtain and renew TLS certificates is **Let's Encrypt** with **Certbot**. Nginx serves HTTPS to visitors, while communication between Nginx and Gunicorn stays on the private Docker network.

---

# Updating Your Application

When you push new code:

```bash
git pull
```

Rebuild and restart:

```bash
docker compose up --build -d
```

Apply migrations if needed:

```bash
docker compose exec web python manage.py migrate
```

Collect static files again if they've changed:

```bash
docker compose exec web python manage.py collectstatic --noinput
```

# What Happens During a Request?

When a user visits:

```text
https://example.com/products/
```

The flow is:

```text
Browser
     │
     ▼
Nginx
     │
     ▼
Gunicorn
     │
     ▼
Django URL Router
     │
     ▼
View
     │
     ▼
PostgreSQL
     │
     ▼
View
     │
     ▼
Gunicorn
     │
     ▼
Nginx
     │
     ▼
Browser
```

For a static asset like:

```text
https://example.com/static/css/style.css
```

The flow is shorter:

```text
Browser
     │
     ▼
Nginx
     │
     ▼
Filesystem
     │
     ▼
Browser
```

Django isn't involved.

# Where to Go Next

You've built a strong foundation. To deepen your Docker skills, I'd recommend learning these topics in order:

1. **Multi-stage Docker builds** (smaller, cleaner images)
2. **Health checks** (`HEALTHCHECK` and Compose healthchecks)
3. **Docker networking** in more depth
4. **Named volumes vs bind mounts** in production
5. **Redis** for caching and sessions
6. **Celery** for background tasks
7. **CI/CD** with GitHub Actions to build and deploy automatically
8. **Container registries** (such as GitHub Container Registry or Docker Hub)
9. **Basic Kubernetes** after you're comfortable with Docker Compose
